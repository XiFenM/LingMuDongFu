## 页面级差异

### 1. 首页 `/`

主要差异：

1. **年轮时间线形态错误**：原型 `首页(修改年轮后).png` 是一个大型年轮盘，时间点从年轮盘四周延伸出来；当前实现是横向枝条时间线，和你的预期不一致。
2. **页面纵向密度偏低**：原型 887×1774 内已经包含 Hero、About、Skills、Featured、Timeline、Latest、Contact、Footer；当前同尺寸 viewport 只到 Featured Projects，后续模块被挤到更下面。
3. Hero 左侧竖向挂签在当前宽度下缺失；原型左侧有明显竖签。
4. 技能树缺少原型里的“树干/枝干”主视觉，现在更像围绕中心年轮徽章的六张卡片。
5. 精选项目卡风格不一致：原型是带夹子/藤蔓/纸张边框的横向纸卡；当前是上图下文的大图卡。

优先修复：**重做 Timeline 为大年轮盘版本，并压缩首页模块高度。**

### 2. 博客列表 `/blog`

主要差异：

1. Hero 高度和内容位置偏大，原型更紧凑。
2. 分类/排序栏在当前 941px 宽度下换行，原型是一行完整工具栏。
3. 精选卡和文章卡尺寸偏大，导致同一高度内文章数量少于原型。
4. 原型在 1672px 内包含分页、Contact 和 Footer；当前下方内容未完整进入同一高度。

优先修复：压缩 Hero 高度、工具栏一行化、缩小 BlogCard 高度/图片宽度。

### 3. 博客详情 `/blog/pytorch-dispatcher`

主要差异：

1. 原型文章主体是大宣纸面板，内容密度更高；当前正文卡片过大、首屏可见内容更少。
2. 原型 Hero 标题更小、更靠左，右侧书房场景露出更多；当前标题更大且换行。
3. TOC/文章信息侧栏位置和尺寸接近，但整体进入页面更靠下。
4. 原型底部的上一篇/下一篇、相关文章、联系区在 1536 高度内可见；当前首屏主要停留在正文开头。

优先修复：降低 Hero 高度、缩小正文字号/卡片 padding、提高文章信息密度。

### 4. 项目列表 `/projects`

主要差异：

1. 这是当前最接近原型的一页。
2. Hero 背景构图与原型不同：当前圆窗更居中偏右，标题区域更大。
3. 项目卡整体接近，但缩略图题材和明暗关系与原型不完全一致。
4. 底部 CTA 在当前 viewport 中不如原型紧凑；原型能看到完整 CTA 与 Footer。

优先修复：轻调 Hero 高度/背景裁切、压缩列表和 CTA 间距。

### 5. 项目详情 `/projects/cuda-softmax`

主要差异：

1. 原型标题是“Triton & CUDA 核心原理整理”，当前数据是“CUDA Softmax Kernel”；内容命名和原型不一致。
2. 原型首屏目录、项目概览、右侧信息卡在更靠上的位置；当前 Hero 高度较大，正文整体下移。
3. 当前技术图示已经补上，但与原型相比尺寸/位置/文字密度仍不同。
4. 右侧纸卡栏在当前截图中更靠右且局部超出对比区域，侧栏布局需收紧。

优先修复：对齐项目详情数据标题/副标题，降低 Hero 高度，收紧三栏布局。

### 6. 404 `/404`

主要差异：

1. 原型 404 数字非常大，且与文案形成强叙事；当前数字较小。
2. 原型主视觉断桥/树洞叙事更居中明显；当前背景更暗，右侧挂轴更突出。
3. 原型提示卡是横向旧纸卡，带大灵萤；当前提示卡较宽但灵萤图形不足。
4. 原型下方树心小门横条和 Contact 卡在同一屏内更紧凑；当前额外插入了 portal section，导致下移。

优先修复：放大 404 数字和提示卡，移除/压缩额外 portal section，恢复大灵萤提示。

### 7. 简历 `/resume`

主要差异：

1. 整体接近度较好，仅次于项目列表。
2. Hero 当前更高，简历内容进入页面更靠下。
3. 原型左侧 profile 卡更窄、更贴左；当前头像卡视觉更大。
4. 原型工作经历、代表项目、荣誉区在 1536 高度内更完整；当前下方项目区域露出较少。

优先修复：压缩 Hero 和 Profile 卡高度，增加右侧内容密度。
