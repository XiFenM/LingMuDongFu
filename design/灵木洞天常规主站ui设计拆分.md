# 《灵木洞天》常规主站 UI 设计拆分

## 1. 设计目标

本文件用于细化《灵木洞天》个人主页中“常规主站”的视觉 UI 设计需求，方便后续进行 Figma 设计、AI 生成 UI 稿、前端组件开发和设计验收。

常规主站的核心目标是：

> 在保证个人主页信息清晰、专业可信的前提下，用“灵木洞天”美学建立独特视觉记忆点，并通过隐藏入口引导少部分访客进入 2D 游戏彩蛋。

---

## 2. 常规主站页面范围

### 2.1 MVP 必做页面

| 页面 | 路径 | 页面类型 | 目标 |
|---|---|---|---|
| 首页 | `/` | 长页面 | 完整展示个人核心信息，是主站最重要页面 |
| 项目详情页 | `/projects/[id]` | 内容详情页 | 展示重点项目的背景、技术难点、成果和复盘 |
| 博客列表页 | `/blog` | 内容列表页 | 展示文章、笔记、专题分类 |
| 博客详情页 | `/blog/[slug]` | 内容阅读页 | 展示具体技术文章或笔记 |
| 404 页面 | `/404` | 异常状态页 | 保持主题一致，提供返回入口 |

### 2.2 建议做但非 MVP 必需页面

| 页面 | 路径 | 页面类型 | 目标 |
|---|---|---|---|
| 项目列表页 | `/projects` | 内容列表页 | 当项目数量较多时集中展示全部项目 |
| 关于页 | `/about` | 个人介绍页 | 当首页关于信息不够时扩展 |
| 简历页 | `/resume` | 简历展示页 | 在线展示简历或提供 PDF 下载 |
| 标签页 | `/tags/[tag]` | 聚合页 | 按技术标签聚合博客和项目 |

### 2.3 特殊入口但不作为常规页面主流程

| 页面 | 路径 | 说明 |
|---|---|---|
| 游戏彩蛋页 | `/game` | 由首页隐藏入口触发，资源延迟加载，不进入常规导航主链路 |

## 3. 视觉定位

### 3.1 风格关键词

- 东方仙侠
- 灵木洞天
- 树屋仙府
- 古树年轮
- 青苔木纹
- 月光纸窗
- 竹简玉简
- 灵萤粒子
- 云雾留白
- 温润、幽静、专业、可读

### 3.2 设计气质

主站不做重度游戏 UI，也不做页游式大光效。它应该更像：

> 一个专业技术人的作品集，被安放在一座安静的树中仙府里。

视觉上应做到：

- 有仙侠意象，但不牺牲阅读效率。
- 有东方美学，但不堆砌传统符号。
- 有沉浸感，但不影响招聘方快速判断能力。
- 有装饰，但组件结构仍然现代、清晰、可复用。

### 3.3 视觉禁区

避免以下方向：

- 大面积金色、红色、紫色光污染。
- 过度页游化按钮和边框。
- 复杂龙凤纹样覆盖信息内容。
- 大量书法字体用于正文。
- 文字区域背景太花，导致阅读困难。
- 每个模块都做强动效，造成疲劳。
- 把所有技术词汇强行翻译成仙侠话术。

## 4. 全站设计系统

## 4.1 页面画布规格

设计稿建议至少包含以下尺寸：

| 设备 | 画布宽度 | 用途 |
|---|---:|---|
| Desktop | 1440px | 主设计稿 |
| Laptop | 1280px | 适配检查 |
| Tablet | 768px | 平板布局 |
| Mobile | 390px | 移动端主设计稿 |

### 4.2 栅格系统

#### Desktop

- 内容最大宽度：1120px 到 1200px。
- 栅格：12 columns。
- 页面左右安全边距：80px 到 120px。
- 模块上下间距：96px 到 144px。
- 卡片间距：24px 到 32px。

#### Tablet

- 内容左右边距：40px。
- 栅格：8 columns。
- 模块上下间距：80px 到 112px。

#### Mobile

- 内容左右边距：20px。
- 栅格：4 columns。
- 模块上下间距：64px 到 88px。
- 卡片改为单列。

## 5. 色彩系统

### 5.1 主色板

| 名称 | 色彩方向 | 使用场景 |
|---|---|---|
| 洞天墨绿 | 深青绿、近黑绿 | 页面主背景、深色区域 |
| 古木深棕 | 深木色 | 卡片底色、分割区域、边框 |
| 青苔绿 | 柔和低饱和绿 | 标签、辅助图形、hover 状态 |
| 灵萤青 | 淡青绿色发光 | 高亮、链接、重点数字、彩蛋入口 |
| 月白 | 温润白，不用纯白 | 正文、标题、主要文字 |
| 宣纸米白 | 暖白、米色 | 浅色卡片、文章阅读背景 |
| 淡金 | 低饱和金色 | CTA、少量强调、年轮节点 |
| 朱砂暗红 | 暗红 | 印章、危险状态、少量装饰 |

### 5.2 页面色彩比例

推荐比例：

- 深色背景：60%。
- 木色 / 青苔 / 中性色：25%。
- 灵光高亮：10%。
- 朱砂 / 淡金强调：5%。

### 5.3 使用原则

- 正文区域必须保证对比度。
- CTA 可以使用灵萤青或淡金，但不要两个同时抢主视觉。
- 大面积背景优先深墨绿、深木色、暗青色。
- 游戏彩蛋入口可以使用更明显的灵萤青发光，但需要克制。

## 6. 字体系统

### 6.1 字体层级

| 层级 | 用途 | 风格建议 |
|---|---|---|
| Display | 首页主标题 | 可使用有东方气质的标题字体或现代衬线字体 |
| H1 | 页面标题 | 稳重、清晰、有气质 |
| H2 | 模块标题 | 可加入轻微装饰性 |
| H3 | 卡片标题 | 现代、清楚 |
| Body | 正文 | 必须高可读，现代无衬线优先 |
| Caption | 标签、说明 | 简洁、低干扰 |
| Decorative | 印章、符文、角标 | 只用于装饰，不承载核心信息 |

### 6.2 字号建议

#### Desktop

| 类型 | 字号 |
|---|---:|
| Hero 主标题 | 56px 到 72px |
| Hero 副标题 | 22px 到 28px |
| 页面 H1 | 44px 到 56px |
| 模块 H2 | 32px 到 40px |
| 卡片标题 | 20px 到 24px |
| 正文 | 16px 到 18px |
| 辅助文字 | 13px 到 15px |

#### Mobile

| 类型 | 字号 |
|---|---:|
| Hero 主标题 | 36px 到 44px |
| Hero 副标题 | 18px 到 22px |
| 模块 H2 | 26px 到 32px |
| 卡片标题 | 18px 到 22px |
| 正文 | 15px 到 17px |

### 6.3 字体使用原则

- 正文不要用书法字体。
- 技术关键词必须清晰可读。
- 标题可以有东方气质，但不能牺牲辨识度。
- 中英文混排要保持基线、字距和行高舒适。

## 7. 图形语言

### 7.1 核心图形母题

| 母题 | 用法 |
|---|---|
| 年轮 | 时间线、模块分割、背景纹理 |
| 树根曲线 | 导航线、技能树分支、页面动线 |
| 竹简 | 博客卡片、说明卡片 |
| 玉简 | 技术笔记、项目高亮 |
| 纸窗 | 卡片背景、文章阅读页侧边装饰 |
| 灵萤 | 彩蛋入口、hover 效果、加载状态 |
| 云雾 | 页面过渡、首屏背景 |
| 符文环 | 技能节点、项目图标容器 |
| 月窗 | 联系区域、页脚视觉 |
| 青苔 | 卡片角落、分割线、装饰边缘 |

### 7.2 图形使用原则

- 图形元素应作为信息结构的辅助，不要成为视觉噪音。
- 每屏最多一个强视觉母题。
- 纹理透明度保持低，避免干扰文字。
- 装饰图形不要与按钮、链接混淆。

## 8. 全站组件清单

### 8.1 基础布局组件

| 组件 | 用途 | 要求 |
|---|---|---|
| AppShell | 页面整体外壳 | 控制背景、导航、页脚 |
| Container | 内容容器 | 统一最大宽度和边距 |
| Section | 页面模块 | 统一标题、间距、锚点 |
| SectionHeader | 模块标题区 | 包含小标题、主标题、说明文字 |
| BackgroundLayer | 背景层 | 控制纹理、雾效、光效 |

### 8.2 导航组件

| 组件 | 用途 | 要求 |
|---|---|---|
| HeaderNav | 顶部导航 | 固定或半固定，滚动后有玻璃/木纹背景 |
| LogoMark | 站点标识 | 可结合灵木、年轮、姓名缩写 |
| NavLink | 导航链接 | hover 有细微灵光或枝叶动效 |
| MobileMenu | 移动端菜单 | 全屏或抽屉式，背景可用深色木纹 |
| ResumeButton | 简历入口 | 清晰可见，不要藏太深 |

### 8.3 按钮组件

| 组件 | 用途 | 要求 |
|---|---|---|
| PrimaryButton | 主要行动按钮 | 用于查看项目、联系我 |
| SecondaryButton | 次级按钮 | 用于阅读博客、下载简历 |
| GhostButton | 弱按钮 | 用于辅助链接 |
| IconButton | 图标按钮 | GitHub、LinkedIn、Email |
| PortalButton | 彩蛋入口按钮 | 隐藏或半隐藏，带灵萤动效 |

按钮视觉要求：

- 圆角不宜过大，建议 12px 到 18px。
- 可以用微弱内发光、细边框、木纹底色。
- 不使用过度厚重的游戏按钮。
- hover 状态需要明确。

### 8.4 卡片组件

| 组件 | 用途 | 要求 |
|---|---|---|
| ProjectCard | 项目卡片 | 标题、摘要、技术栈、亮点、链接 |
| BlogCard | 博客卡片 | 标题、摘要、日期、标签 |
| SkillCard | 技能分类卡 | 技能说明、关联项目 |
| TimelineCard | 经历节点 | 时间、地点、角色、说明 |
| StatCard | 数据指标 | 突出数字和结果 |
| QuoteCard | 引言卡 | 展示个人理念或项目总结 |

卡片视觉要求：

- 背景可以是深木色、半透明墨绿或宣纸浅色。
- 边框可使用细线、年轮纹理、竹简边缘。
- 卡片信息结构必须现代，不做复杂卷轴排版。
- hover 可轻微上浮，或边缘出现灵光。

### 8.5 标签与徽章组件

| 组件 | 用途 | 要求 |
|---|---|---|
| TechTag | 技术栈标签 | CUDA、PyTorch、Triton 等 |
| CategoryPill | 分类标签 | Blog 分类、项目类型 |
| StatusBadge | 状态标记 | Featured、WIP、Open Source |
| HighlightBadge | 亮点标签 | Performance、Distributed、Kernel |

视觉要求：

- 形态像小玉片、叶片或竹签即可，不要太拟物。
- 保持文本清晰。
- 同一页面标签颜色不要过多。

### 8.6 内容展示组件

| 组件 | 用途 | 要求 |
|---|---|---|
| MarkdownRenderer | 博客正文 | 支持代码块、表格、引用、图片 |
| CodeBlock | 代码展示 | 深色主题，与整体风格统一 |
| TOC | 文章目录 | 桌面端可固定在侧边 |
| DataMetric | 指标展示 | 适合项目详情中的性能数字 |
| LinkPreview | 外链预览 | GitHub、Demo、论文等 |

### 8.7 特效组件

| 组件 | 用途 | 要求 |
|---|---|---|
| FireflyParticles | 灵萤粒子 | 数量少，性能友好 |
| MistLayer | 云雾层 | 低透明度慢速移动 |
| TreeRingPattern | 年轮纹理 | 背景或分割符 |
| GlowOrb | 灵光球 | 彩蛋入口、重点节点 |
| ScrollReveal | 滚动出现动画 | 克制、统一节奏 |

## 9. 首页 UI 拆分

首页是常规主站最重要页面，应作为第一优先级设计。

### 9.1 首页模块顺序

推荐顺序：

1. 顶部导航 Header。
2. Hero 首屏。
3. 关于我 About Preview。
4. 技能树 Skill Tree。
5. 精选项目 Featured Projects。
6. 经历 / 年轮时间线 Timeline。
7. 博客 / 笔记精选 Blog Preview。
8. 联系方式 Contact。
9. 页脚 Footer。
10. 隐藏彩蛋入口 Hidden Portal。

---

## 9.2 Header 顶部导航

### 内容元素

- Logo / 姓名标识。
- 导航链接：About、Skills、Projects、Blog、Contact。
- 外部链接图标：GitHub、LinkedIn，可选。
- Resume 按钮，可选。
- 移动端菜单按钮。

### 视觉要求

- 初始状态可透明，叠在 Hero 上。
- 滚动后变为半透明深色背景，带轻微模糊或木纹质感。
- 高度建议 72px 到 84px。
- Logo 可以使用“年轮 + 姓名缩写”的组合。
- 导航 hover 可出现细小枝芽线、灵光点或下划线。

### 状态要求

- 默认状态。
- 滚动吸顶状态。
- 当前 section 激活状态。
- 移动端展开状态。

---

## 9.3 Hero 首屏

### 内容元素

- 姓名 / 英文名。
- 职业定位。
- 一句话介绍。
- 关键词标签。
- 主要 CTA：查看项目。
- 次要 CTA：阅读博客 / 联系我 / 下载简历。
- 背景主视觉。
- 隐藏灵萤入口，可选。

### 建议文案结构

```text
姓名 / Name
AI Framework Engineer · GPU Computing · Distributed Systems

我关注 AI 框架、GPU 计算和大模型系统优化，喜欢把复杂系统拆解成可观察、可优化、可复用的工程结构。

[查看项目] [阅读博客]
```

### 视觉构图

Desktop 推荐：

- 左侧 50% 到 60% 放文字内容。
- 右侧 40% 到 50% 放灵木洞天主视觉。
- 背景可出现巨大树洞、年轮、月光、灵萤。
- 首屏高度建议 760px 到 900px。

Mobile 推荐：

- 文案在上，主视觉简化为背景或顶部插画。
- CTA 纵向排列或自动换行。
- 隐藏入口弱化，不影响首屏操作。

### 视觉资产要求

- 一张 Hero 主视觉插画。
- 灵萤粒子若干。
- 背景纹理：深色木纹、雾、年轮。
- 可选：月窗、树根边框。

### 验收要求

- 5 秒内能看懂站主是谁、做什么、擅长什么。
- CTA 清晰。
- 背景不干扰文字。
- 不依赖游戏资源。

## 9.4 About Preview 关于我模块

### 内容元素

- 模块标题。
- 1 到 2 段个人介绍。
- 3 个特质或方向标签。
- 可选：头像或抽象形象。
- 可选：教育 / 当前状态摘要。

### 视觉构图

推荐两栏：

- 左侧：竹简 / 洞天铭文风格的介绍卡片。
- 右侧：个人关键词、教育或方向信息。

### 组件

- SectionHeader。
- AboutCard。
- TraitBadge。
- MiniProfilePanel。

### 视觉要求

- 不要把介绍写成古风人设。
- 可以在模块角落加入竹叶、青苔、淡淡年轮。
- 正文必须可读。

---

## 9.5 Skill Tree 技能树模块

### 内容元素

技能分类建议：

- AI Framework。
- PyTorch Internals。
- CUDA / GPU Computing。
- Distributed Training。
- LLM Systems。
- Backend / Tooling。
- Statistics / ML。

每个技能节点包含：

- 技能名称。
- 熟悉程度或说明。
- 关联项目 / 文章。

### 视觉构图

Desktop 推荐：

- 中间为一条灵木主干。
- 左右展开技能分支。
- 每个分支末端是技能节点。
- 节点可用灵光、叶片、玉片表示。

Mobile 推荐：

- 改为分组卡片列表。
- 不强行保留复杂树状布局。

### 组件

- SkillTree。
- SkillBranch。
- SkillNode。
- SkillTooltip。
- SkillLegend。

### 交互要求

- hover 技能节点显示简短说明。
- 点击节点可跳转相关项目或文章。
- 当前暂无关联内容时不显示跳转状态。

### 视觉要求

- 技能树不能做成难以阅读的装饰图。
- 技术名称必须一眼可见。
- 节点数量控制在 20 到 35 个以内。

---

## 9.6 Featured Projects 精选项目模块

### 内容元素

每个项目卡片包含：

- 项目名称。
- 项目一句话摘要。
- 技术栈标签。
- 2 到 3 个亮点。
- 指标数据，可选。
- GitHub / Demo / 查看详情按钮。

### 推荐展示数量

首页展示 3 到 6 个精选项目。

### 视觉构图

Desktop 推荐：

- 重点项目使用大卡片，占据 2 columns。
- 其他项目使用中卡片网格。
- 背景可表现为“炼丹房”的案台、丹方、法器匣，但卡片本身保持现代。

Mobile 推荐：

- 单列卡片。
- 亮点信息折叠或压缩。

### 组件

- ProjectGrid。
- ProjectCard。
- ProjectFeatureList。
- TechTagGroup。
- ProjectMetric。
- ProjectLinkGroup。

### 视觉要求

- 卡片 hover 可出现淡淡灵光边缘。
- Featured 项目可加“精选”角标，但不要太游戏化。
- 技术栈标签应统一尺寸。

---

## 9.7 Timeline 年轮时间线模块

### 内容元素

- 教育经历。
- 工作经历。
- 项目里程碑。
- 重要成长节点。

每个节点包含：

- 时间。
- 标题。
- 角色 / 组织。
- 简短说明。
- 可选关联链接。

### 视觉构图

Desktop 推荐：

- 以年轮弧线或树干纵向纹理作为时间线。
- 节点像年轮上的发光刻痕。
- 左右交错排列卡片。

Mobile 推荐：

- 竖向时间线。
- 每个节点单卡片。

### 组件

- Timeline。
- TimelineNode。
- TimelineCard。
- YearMarker。

### 视觉要求

- 时间顺序清晰。

## 14. 隐藏彩蛋入口 UI 细化

### 14.1 入口位置

1. Hero 区域的灵萤入口。
2. Footer 区域的文字入口。

### 14.2 Hero 灵萤入口

交互：

- 点击后打开进入确认弹窗。

```text
这只灵萤似乎想带你去哪里。
```

### 14.3 Footer 文字入口

视觉：

- 像普通页脚文案，但带轻微灵光。

文案：

```text
树心深处，似乎还有一扇微微发光的小门。
```

### 14.4 进入确认弹窗

内容：

- 标题：进入灵木洞天？
- 描述：这是隐藏的 2D 探索彩蛋，可以随时返回主页。
- 按钮：进入洞天 / 还是看看主页。

## 15. 响应式规则

### 15.1 Desktop

- 保留完整视觉元素。
- 技能树可以使用复杂布局。
- 项目卡片可以网格布局。
- TOC 可固定在文章侧边。

### 15.2 Tablet

- 减少装饰元素数量。
- 技能树由复杂树状转为简化树状或双列卡片。
- 项目卡片两列。
- 导航可保留普通导航或切换汉堡菜单。

### 15.3 Mobile

- 页面以阅读效率为主。
- Hero 主视觉弱化。
- 技能树改为分组列表。
- 项目卡片单列。
- 时间线单列。
- 动效显著减少。
- 彩蛋入口保留 Footer 文字入口。

## 16. 动效规范

### 16.1 动效原则

- 轻、慢、少。
- 动效服务于空间感和引导，不服务于炫技。
- 不影响文本阅读。
- 提供减少动态效果适配。

### 16.2 推荐动效

| 动效 | 使用场景 | 要求 |
|---|---|---|
| 灵萤漂浮 | Hero、Footer、彩蛋入口 | 数量少，速度慢 |
| 云雾流动 | 背景层 | 透明度低 |
| 卡片上浮 | hover | 位移 2px 到 6px |
| 边缘灵光 | hover / active | 发光克制 |
| 滚动出现 | Section 入场 | 统一节奏 |
| 年轮线条绘制 | Timeline | 可选，不能太花 |

### 16.3 禁止动效

- 大量闪烁。
- 自动快速移动的大面积背景。
- CTA 按钮持续强烈跳动。
- 会导致布局抖动的 hover。
- 页面加载时过长动画。

## 20. 设计验收标准

### 20.1 信息验收

- 首页不进入游戏也能完整了解站主核心信息。
- 首屏 5 秒内能明确看懂职业定位。
- 项目和技能模块是页面视觉重点之一。
- 联系方式清晰可达。

### 20.2 视觉验收

- 全站风格统一为“灵木洞天”。
- 不出现页游式廉价光效。
- 不出现信息被装饰压住的情况。
- 常规主页看起来仍然专业可信。
- 东方元素点到为止，有记忆点但不过量。

### 20.3 组件验收

- 主要组件都有 desktop 和 mobile 适配方案。
- 按钮、卡片、标签、导航状态完整。
- 组件可以复用，不为每个模块单独做一套风格。

### 20.4 可用性验收

- 页面导航清晰。
- CTA 清晰。
- 移动端阅读舒适。
- 所有链接和按钮有明确交互反馈。
- 彩蛋入口不会误导用户必须进入游戏。

---

## 21. 总结

常规主站的 UI 设计应遵循一个核心原则：

> 专业信息是骨架，灵木洞天是皮肤，隐藏游戏是树心深处的小门。

设计时不能为了仙侠风格牺牲作品集的效率，也不能为了专业感抹掉主题特色。最理想的结果是：普通访客能快速理解你的能力，细心访客又会被这座安静的树中仙府吸引，继续探索更深一层的内容。
