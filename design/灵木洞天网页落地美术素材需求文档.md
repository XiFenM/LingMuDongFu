# 《灵木洞天》网页落地美术素材需求文档

版本：v0.1  
范围：首页、博客列表页、博客详情页、项目列表页、项目详情页、404 页面、简历页  
目标：在不直接开始绘制素材的前提下，拆解这些页面若要高还原落地，需要提前准备、生成或委托绘制的美术素材。

---

## 1. 总体判断

“灵木洞天”不是普通深色网站换皮，它依赖一套完整的视觉资产系统：大场景插画、可复用纹理、纸质/木质卡片边框、灵萤/云雾/光晕特效、项目缩略图、博客水墨插图、月窗茶室页脚等。代码可以实现布局、交互、文本、渐变、阴影和大部分组件状态，但无法凭 CSS 完整还原这些设计图里的树洞空间、月窗、书房、藤蔓、灯笼、纸张旧化、项目科技插画等视觉细节。

落地策略建议分为三层：

1. **P0 基础美术资产**：保证所有页面风格统一且视觉完成度足够高。包括大背景、纹理、Logo、卡片边框、灵萤、页脚联系区、项目缩略图。
2. **P1 页面专属增强资产**：让每个页面有自己的主题空间。包括博客藏书阁、项目工坊、项目研究室、404 未成树洞、简历书桌等专属 Hero 插画。
3. **P2 动效和细节资产**：用于更接近设计图的“灵气”。包括分层云雾、灯笼光晕、月光遮罩、可动画的藤蔓/灵萤、特殊 hover 光效贴图。

## 2. 素材格式与工程交付约定

### 2.1 推荐格式

| 素材类型 | 推荐格式 | 说明 |
|---|---|---|
| 大场景背景 | AVIF / WebP，保留原图 PSD/Figma/分层源文件 | 页面 Hero、页脚、404 等主视觉。 |
| 可复用纹理 | WebP / PNG，尽量可平铺 | 木纹、纸纹、年轮、青苔噪声、雾层。 |
| 装饰边框 / 图标 / 符文 | SVG 优先，复杂半透明用 PNG | 保证缩放清晰，方便主题色控制。 |
| 卡片缩略图 | WebP / AVIF | 项目、博客、证书、相关内容封面。 |
| 透明装饰物 | PNG / WebP with alpha | 藤蔓、青苔、灯笼、灵萤、纸签、角标。 |
| 粒子 / 光效 | CSS 渐变 + SVG/PNG 小贴图 | 灵萤、光点、月光、法阵脉冲。 |
| Open Graph 图 | PNG / JPG / WebP | 社交分享用，1200×630。 |

### 2.2 尺寸建议

| 层级 | Desktop | Mobile | 备注 |
|---|---:|---:|---|
| Hero 主视觉宽图 | 2880×1200 或 2880×1400 | 900×1200 | 按页面提供安全裁切区。 |
| 中型 Section 背景 | 2400×700 | 900×700 | 技能树、时间线、CTA 等。 |
| 项目卡片缩略图 | 1200×675 | 800×450 | 16:9，方便列表和详情复用。 |
| 博客卡片插图 | 800×450 或 640×360 | 640×360 | 可做水墨小图，不必张张大场景。 |
| 纸质卡片纹理 | 1024×1024 可平铺 | 同左 | 可叠加 CSS border。 |
| 木纹 / 年轮纹理 | 1024×1024 或 2048×2048 | 同左 | 建议平铺或低透明叠层。 |
| 装饰角 / 边框 | SVG 或 256×256 alpha | 同左 | 适配 1x/2x/3x。 |
| Logo | SVG + PNG 512 | 同左 | 深浅底都要能用。 |

### 4.1.2 可用代码替代的部分

- 按钮、标签、导航线、卡片 hover：CSS + SVG 边角足够。

- 技能树文字框：HTML/CSS 绘制，树枝本体建议用插画/SVG。

### 4.4.1 项目封面统一规范

- 比例统一 16:9。
- 主题色统一深绿、蓝绿、金色点光。
- 不出现真实品牌 Logo，避免版权与风格不统一。
- 每张封面要有“技术主题可辨识物”：芯片、网络、树、数据流、方阵、卷轴等。
- 封面只做氛围，不承载关键文字，文字由前端组件渲染。

## 7. 美术生产顺序建议

建议不要按页面一张张画完，而是按“可复用程度”倒排：

1. **先做视觉基础包**：Logo、纹理、边框、标签、灵萤、图标。
2. **再做全站复用场景**：月窗茶室 Contact/Footer。
3. **再做页面 Hero 背景**：首页、博客、项目列表、项目详情、404、简历。
4. **再做内容缩略图**：项目封面 6–12 张，博客插图 3–8 张。
5. **最后做特效层**：云雾、光晕、前景遮挡、hover 流光、Portal。

这样做的好处是，即使页面大背景没全部完成，前端也能先用统一纹理、卡片和图标跑起来，不会卡在“等一张神图”的树洞门口。

## 8. 页面还原风险与取舍建议

### 8.1 高风险素材

- **首页 Hero 主视觉**：信息密度高，必须预留文字安全区。若一张图画死，移动端裁切容易翻车。
- **技能树主干**：若用一整张图，文字节点难以响应式。建议树干/枝干用 SVG，技能文字用 DOM。
- **年轮时间线**：可以是独立插画，但节点和文本要前端渲染。
- **项目详情技术图示**：最好用 SVG/HTML 混合，不要把代码和表格画进图片。
- **博客详情文章纸张**：必须支持任意长度，不可用固定高度背景图。
- **404 主视觉**：需要左侧暗区，否则 404 文案会被背景吃掉。
- **简历头像**：真实照片、AI 头像、插画头像需提前定方向，避免之后风格割裂。

### 8.2 推荐取舍

| 设计图元素 | 建议实现方式 |
|---|---|
| 大场景背景 | 美术插画，分层更佳。 |
| 文字内容、代码、表格 | 前端真实 DOM，不画进图里。 |
| 卡片边框 | 9-slice PNG/SVG + CSS。 |
| 标签、按钮 | CSS 为主，纹理/边角辅助。 |
| 技术图示 | SVG/HTML 混合。 |
| 粒子、微光 | CSS/Canvas + 少量贴图。 |
| 复杂水墨小插图 | 图片素材。 |
| 图标 | SVG icon set。 |

## 10. 结论

这组页面的美术需求可以概括成一句话：**先建立一套“灵木洞天视觉积木”，再为每个页面配置一间不同功能的树中房间。**

真正不可替代的美术工作集中在三块：

1. **场景插画**：决定页面第一眼是否有“洞天感”。
2. **纹理与边框系统**：决定组件是否统一，而不是每页像不同模型生成的孤岛。
3. **内容缩略图与水墨插图**：决定项目和博客列表是否有持续扩展能力。

先完成 P0 素材包，前端就可以开始搭建大部分页面；P1/P2 再逐步补灵气、补分层、补动效。不要把所有视觉都压进一张大图，最理想的工程形态是：**场景图做氛围，组件图做质感，文字和交互交给代码。**
